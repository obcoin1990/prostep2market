# Key UI patterns from RESEARCH.md (Pattern 1: Multi-Step Assessment Wizard):

AssessmentWizard component structure:
- currentStep state (0-4 for 5 sections)
- answers state (AnswerMap)
- isComplete state
- profile state
- Section navigation: back/next buttons
- Section validation: check all questions answered before allowing next

ProgressBar:
- current: number (1-5)
- total: number (5)
- sectionName: string from sections object

QuestionCard:
- Receives question, value, onChange
- Delegates to type-specific component based on question.type
</context>

Create src/components/trader-dna/assessment-wizard.tsx:

1. 'use client' directive

2. Imports: useState, questions, sections from data, calculateProfile from scoring, ProgressBar, QuestionCard

3. State:
   - currentStep: number (0-4)
   - answers: AnswerMap (Record<string, number | string>)
   - isComplete: boolean
   - profile: TraderProfile | null

4. Derived values:
   - sectionKeys: Object.keys(sections) as SectionKey[]
   - currentSection: sectionKeys[currentStep]
   - sectionQuestions: questions.filter(q => q.section === currentSection)

5. Functions:
   - handleAnswer(questionId, value): update answers state
   - isSectionComplete(): returns true if all sectionQuestions have answers in answers
   - handleNext(): validates section complete, then increments step or calculates profile
   - handleBack(): decrements step

6. Render:
   - If isComplete && profile: return placeholder for ProfileSummary (will be Plan 04)
   - Otherwise: render ProgressBar + map sectionQuestions to QuestionCard + Back/Next buttons
   - Next button disabled if !isSectionComplete()
   - Final step button says "Generate Profile"

7. Add proper TypeScript types throughout
  </action>
