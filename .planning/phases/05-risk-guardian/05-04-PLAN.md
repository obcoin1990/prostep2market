## Pause Mode Behavior (from CONTEXT.md)

```typescript
interface PauseMode {
  active: boolean;
  startedAt?: Date;
  durationMinutes?: number;  // 15, 30, 60, or null (manual resume)
  reason?: 'user_initiated' | 'system_suggested';
}
```

When activated:
1. Display overlay on all trading screens
2. Block trade entry forms (read-only mode)
3. Show countdown timer if duration set
4. User can manually resume before timer ends
5. System can suggest pause (after fatigue alert) but user decides
</context>

  <files>src/app/api/guardian/pause/route.ts, src/app/api/guardian/resume/route.ts</files>
  <action>
Create two API route files under `src/app/api/guardian/`:

**src/app/api/guardian/pause/route.ts** — POST handler:
- Get authenticated user
- Accept body: `{ durationMinutes?: number, reason?: 'user_initiated' | 'system_suggested' }`
- durationMinutes: validate it is 15, 30, 60, or null (manual)
- Insert into `pause_mode` table (create table in migration 005):
  ```sql
  CREATE TABLE pause_mode (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    active BOOLEAN DEFAULT TRUE,
    duration_minutes INT,
    reason TEXT DEFAULT 'user_initiated',
    started_at TIMESTAMPTZ DEFAULT NOW()
  );
  ```
- Return: `{ pauseMode: { id, active, durationMinutes, startedAt, reason } }`

**src/app/api/guardian/resume/route.ts** — POST handler:
- Get authenticated user
- Update `pause_mode` table: `UPDATE ... SET active = false WHERE user_id = userId AND active = true`
- Return: `{ resumed: true }`

Also add to migration 005: `CREATE INDEX idx_pause_mode_user_active ON pause_mode(user_id, active)`.
Add RLS policies for `pause_mode` table.

Import Supabase from `~/lib/supabase/client`.
