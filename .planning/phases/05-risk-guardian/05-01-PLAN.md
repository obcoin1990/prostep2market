## Detection Rules (locked in CONTEXT.md)

| Alert Type | Detection Rule | Severity |
|------------|---------------|----------|
| Revenge Trading | 3+ trades within 30 min after 2+ consecutive losses | critical |
| Fatigue | Session > 2 hours OR > 50 trades in single session | warning |
| Risk Escalation | Current exposure > 150% of profile normal | critical |
| Emotional Instability | 5+ trades with emotional_state = 'frustrated' or 'fearful' in 24h | warning |
| Exposure Warning | Lot size > 130% of historical average | warning |
| Overtrading | > 8 trades in 4-hour window | warning |

## RiskGuardianSettings Interface (from CONTEXT.md)

```typescript
interface RiskGuardianSettings {
  userId: string;
  maxSessionDuration: number;       // minutes, default 120
  maxTradesPerSession: number;        // default 50
  maxTradesPerWindow: number;         // window in 4h, default 8
  exposureMultiplier: number;         // % above normal, default 130

  revengeTradingAlertEnabled: boolean; // default true

}
```

Create `src/lib/risk-guardian/rules/` directory and implement all detection rules as individual modules. Each file exports a single detection function that returns an Alert or null.

**revenge.ts** — GRDN-02:
- Find 2+ consecutive losses in recentTrades
- Look for 3+ trades within 30 minutes after the last consecutive loss
- Return critical Alert with revenge_trading type and suggestedAction: 'Take a 15-minute break before continuing.'
- tradeIds should include the follow-up trades

**fatigue.ts** — GRDN-03, GRDN-06:
- Check if session duration (Date.now() - sessionStart) > settings.maxSessionDuration minutes
- Check if recentTrades.length > settings.maxTradesPerSession
- Return warning Alert if either threshold exceeded
- title: 'Trading Fatigue Warning'
- suggestedAction: 'Consider taking a break.'

**emotional.ts** — GRDN-05:
- Count trades where emotionalState === 'frustrated' OR emotionalState === 'fearful' in last 24 hours
- Use triggeredAt from each trade to filter 24h window
- If count >= settings.emotionalInstabilityThreshold, return warning Alert
- title: 'Emotional Instability Detected'

**exposure.ts** — GRDN-04, GRDN-07:
- detectRiskEscalation: currentExposure > (historicalAverage * settings.exposureMultiplier) returns critical Alert
- detectExposureWarning: any trade lotSize > (historicalAverage * settings.exposureMultiplier) returns warning Alert
- Use settings.exposureMultiplier value directly

**overtrading.ts**:
- Count trades within a 4-hour sliding window
- If count > settings.maxTradesPerWindow, return warning Alert
- Use date-fns for time window calculations

Each function signature: `(input: DetectionInput) => Alert | null`

Import DetectionInput from `../../types/guardian`. Do NOT import from other rule files.

Create `src/lib/risk-guardian/detector.ts` as the central orchestrator that coordinates all rule evaluations.

Export the following functions:

**`detectAlerts(input: DetectionInput): Alert[]`**
- Imports and calls all 5 rule functions (revenge, fatigue, emotional, exposure, overtrading)
- Collects results, filters nulls, returns Alert[]
- Handles deduplication: skip if same alert type already created within cooldown period
- Cooldown: same alert type needs 5-minute gap before re-triggering

**`runFullCheck(userId: string, settings: RiskGuardianSettings): Promise<Alert[]>`**
- Fetches recentTrades from Supabase (last 50 trades, last 24 hours)
- Calculates historicalAverage from last 50 trades' lot sizes
- Determines sessionStart from session-tracker
- Builds DetectionInput and calls detectAlerts
- Returns Alert[]

**`createAlert(params: Partial<Alert> & { type: AlertType; severity: Severity; title: string; message: string; suggestedAction: string }): Alert`**
- Generates a new Alert object with id (crypto.randomUUID()), userId, triggeredAt (now), acknowledged: false

Import rules from `./rules/revenge`, `./rules/fatigue`, `./rules/emotional`, `./rules/exposure`, `./rules/overtrading`.
Import Supabase from `~/lib/supabase/client`.
